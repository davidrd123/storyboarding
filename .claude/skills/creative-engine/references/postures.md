# Control Postures (adapted DAYDREAMER goal types)

A posture is how the CONTROLLER is thinking; an operator is which mutation it
applies. Postures are **routed by session state — never drawn randomly**
(randomly imposed postures become metaphorical or arbitrary; that distinction
is Mueller's architecture).

| Posture | What it is | Route into it when |
|---|---|---|
| **ROVING** | associative exploration: run cascades from surprising fragments without a pressing concern; collect, don't develop | session start after store build; after a frame exhausts; when the niche map has unexplored regions and no live thread |
| **REVERSAL** | negate or undo a premise: "how would this success have failed?" / "what if the opposite were true?" | a seed feels inevitable/too smooth; Gate 1 stress-pass on every portfolio candidate |
| **REPERCUSSIONS** | follow consequences outward: run the rule's physics into neighboring scenes, characters, days | a seed passed Gate 0 with energy to spare; before writing the LADDER |
| **REHEARSAL** | simulate the concept through shots: walk the beat frame by frame at one-second granularity; harvest what the simulation surfaces | a concept reads well but nobody has SEEN it yet; pre-FRAME check |
| **RECOVERY** | repair a failed-but-fertile candidate: mine dead seeds for transplantable parts ("I'm on a horse" was scavenged from a rejected script) | the nursery holds flagged/unbridged seeds with strong fragments; end-of-session sweep before anything is abandoned |

Dropped from Mueller for v0 (deliberately, not silently): REVENGE and
RATIONALIZATION — they belong to his emotional architecture and turn
arbitrary when imposed on commercial ideation. RATIONALIZATION's useful
ghost survives as Error Adoption at Gate 3 (honour thy error).
