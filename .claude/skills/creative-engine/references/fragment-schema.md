# Fragment Store — schema and protocol

The engine is only as surprising as its indices. Ordinary tags ("candy",
"fun", "transformation") make coincidence retrieval reproduce ordinary
thought. A good retrieval happens because a lampshade, a parachute, a fruit
rind, a theatrical curtain, and a shedding insect all share "thin covering"
+ "release" + "inside becoming outside" — not because they all resemble the
product.

## Store format

```json
{"fragments": [
  {"id": "f001",
   "text": "one compact, concrete fragment — an image, fact, or mechanism",
   "source": "where it came from (script p.2 / research/03 §2 / MJ board G3)",
   "tags": {
     "physical-operation": ["peel"],
     "material-behavior": ["rind"],
     "emotional-fantasy": ["escape"]
   }}
]}
```

## The ten index families (fixed — values controlled per store)

| Family | Starter values |
|---|---|
| physical-operation | peel, split, unroll, laminate, inflate, dissolve, fold, shed, unlock, decant |
| material-behavior | rind, cellophane, rubber, fabric, foam, glass, wax, clay, paper, syrup |
| spatial-relation | inside-out, nested, suspended, orbiting, compressed, layered, mirrored, threshold |
| emotional-fantasy | escape, recognition, power, intimacy, belonging, relief, mischief, awe |
| social-configuration | alone, witnessed, paired, crowd, convergence, broadcast |
| temporal-behavior | gradual, instantaneous, recursive, delayed, looping, rehearsed, already-over |
| formal-medium | live-action, cel-animation, clay, diagram, toy-theater, macro, security-cam |
| transition-affordance | mouth-portal, match-cut, wipe, rotation, rupture, peel-reveal, reflection |
| product-fact | layers, fruit-form, texture-contrast, peel-or-eat-choice, slow-ritual, sticky |
| productive-contradiction | controlled-chaos, tactile-spectacle, lucid-weirdness, polite-violation |

Extend value lists per store, but keep values **lowercase, hyphenated,
reused across fragments** — a value used once can never be a coincidence.
`fragments.py validate` warns when a value indexes >30% of the store
(near-useless as a cue) and requires ≥3 families per fragment.

## Sourcing rules (anti-homogenization)

1. Fragments come from EXTERNAL material only: script beats, product physics,
   research-file examples, mood boards, reference films, found references.
   Never free-generated by the model that will retrieve them.
2. 40–100 compact fragments for a session-scale store.
3. **Blind tagging:** the tagging pass runs WITHOUT the brief in context —
   tag the fragment for what it is, not for what the job needs. (If the
   brief leaks into tagging, the "independent cues" aren't independent and
   coincidence retrieval degenerates into similarity retrieval.)
4. The store is usage-unbiased, forever. Retrieval order may consider
   recency/concern strength; the portfolio records preferences; **neither
   ever rewrites fragment indices.** A never-traversed path must stay
   exactly as findable as a hundred-times-traversed one.

## Retrieval tuning (probe before trusting)

Probe finding (synthetic stores, 40–100 fragments): threshold 1 floods,
thresholds 3–4 mostly retrieve nothing; the usable band is narrow.

- With ~6 activated cues: base threshold 2 (serendipity mode: don't lower to
  1 — it floods; widen cues by 1–2 instead).
- With ~8 activated cues: base threshold 3 (strict, occasionally empty —
  that's fine), serendipity 2 (generative).
- Always run `fragments.py probe --store STORE` after building a real store;
  pick the threshold where the median retrieval is 2–6 and zero% < 35.

## Activating cues from a concern

Translate the concern (loop step 1) into cue values across DIFFERENT
families — e.g. Tina/car: `peel, escape, alone, instantaneous,
peel-or-eat-choice, compressed, rind, mouth-portal`. Cues spanning families
is what makes convergence mean something.

## SEED / WILD (human injection)

- **SEED:** a human-authored fragment enters the store normally but its
  values join the active cues (privileged initial activation).
- **WILD:** human material bypasses retrieval entirely, goes straight to
  displacement/development — but still gets lineage and a later bridge.
- Neither receives any automatic fitness bonus.
